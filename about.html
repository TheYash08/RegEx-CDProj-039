A complete Regular Expression Engine implementing all five phases of compiler theory — from Lexical Analysis all the way to DFA Minimization — built entirely in vanilla JavaScript with zero dependencies. Includes visual state diagrams for NFA, DFA and Minimized DFA.

PHASES
Each Phase Explained
1

Lexer — Tokenization

LEXICAL ANALYSIS

The Lexer reads the regex pattern character by character and converts it into a typed token stream. Each character is classified: CHAR, STAR (*), PLUS (+), QUES (?), OR (|), LPAREN, RPAREN, DOT (.). This is the first phase of every compiler.

Regular ExpressionsTokenizationLexical AnalysisCharacter Scanning
2

Parser — Abstract Syntax Tree

SYNTAX ANALYSIS · RECURSIVE DESCENT

The Parser builds an AST using Recursive Descent Parsing. Operator precedence: quantifiers (*, +, ?) bind tightest, then concatenation, then alternation (|). Explicit CONCAT tokens are inserted between adjacent operands before parsing begins.

Recursive DescentAbstract Syntax TreeOperator PrecedenceCFG
3

NFA Builder — Thompson's Construction

NON-DETERMINISTIC FINITE AUTOMATON · VISUAL GRAPH

Thompson's Construction recursively converts each AST node into a small NFA fragment joined via ε-transitions. Each operator has its own NFA construction pattern. The result has exactly one start and one accept state. A visual graph diagram is drawn showing all states and transitions.

Thompson's ConstructionNFAε-TransitionsState Diagrams
4

DFA Converter — Subset Construction

DETERMINISTIC FINITE AUTOMATON · VISUAL GRAPH

Subset Construction converts the NFA to a DFA. Each DFA state represents a set of NFA states reachable via ε-closure. A DFA state is accepting if any of its NFA states are accepting. A visual graph is drawn showing the deterministic automaton.

Subset ConstructionDFAε-ClosurePowerset Construction
5

DFA Minimization — Hopcroft's Algorithm

STATE MINIMIZATION · EQUIVALENCE PARTITIONING · VISUAL GRAPH

Hopcroft's Algorithm minimizes the DFA by partitioning states into equivalence classes. States that cannot be distinguished by any input string are merged. The result is the smallest DFA accepting the same language. A visual graph shows the minimized automaton with merged state labels.
